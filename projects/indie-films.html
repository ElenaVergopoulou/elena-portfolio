Overview

This project involved designing the information architecture and key wireframes for an Indie Film Platform that merges film discovery, crowdfunding, and events. Unlike existing platforms, this concept aimed to support independent creators by offering visibility, community-building tools, and financial support features under one roof.

The project focused on four core areas:

  • Discovering indie films
  • Viewing and supporting crowdfunding campaigns
  • Exploring filmmaker profiles and film events
  • Ensuring a seamless, intuitive user journey supported by strong IA

Through interviews, competitive benchmarking, tree testing, and iterative wireframing, I developed a platform structure that reflected both filmmaker and viewer needs.

Early-stage domain model and sitemap showing platform structure across films, campaigns, profiles, and events.

Project Goals

  1. Design a clear and intuitive information architecture that reflects how real users would browse and support indie films
  2. Merge film discovery and crowdfunding into a unified user journey
  3. Support diverse user needs ‚Äì from casual browsers to active backers and filmmakers
  4. Ensure content scalability with a modular navigation structure that supports campaigns, profiles, and events

My Role

This was an individual project where I:

  • Conducted interviews with a filmmaker, film student, and cinephile
  • Mapped out the domain model
  • Performed card sorting and tree testing to validate structure and labels
  • Designed the sitemap using hierarchical grouping and user logic
  • Created task-based user journeys reflecting realistic platform use cases
  • Developed four annotated wireframes for core pages: Discover Films, Film Details, Campaigns, and Events
  • Conducted evaluation sessions to test terminology clarity, navigation structure, and result display formats

My responsibilities spanned the full IA process‚Äîfrom early research and stakeholder interviews, to defining domain relationships, testing navigation patterns, and iterating wireframes based on user feedback.

Methodology

Interviews & Domain Analysis

I interviewed three relevant stakeholders to identify real needs and mental models. These sessions revealed the importance of events as community-building and discovery tools ‚Äî leading me to promote Events to a main navigation category.

Competitive Benchmarking

I analyzed platforms like IMDb, Letterboxd, and Kickstarter to understand how established services structure content and guide users. Key takeaways included:

  • Letterboxd: Strong filtering and search mechanisms
  • Kickstarter: Clear crowdfunding communication and progress tracking
  • IMDb: Dense but logically structured content layouts

IA Techniques

  • Domain Model: Structured around core entities like films, campaigns, users, events, and profiles
  • Card Sorting & Tree Testing: Used to test labeling clarity and hierarchy for key user actions
  • Sitemap: Created using user-centered design principles and clarity heuristics (Morville & Rosenfeld)
  • User Journeys: Mapped realistic goal-based scenarios to define how different users would navigate through discovery, support, and participation flows

Findings

Key usability insights gathered from interviews, card sorting, and tree testing included:

  • Browsing preferences: Users preferred discovering films through timeframes (e.g. ‚ÄúRecently Added,‚Äù ‚ÄúClassics‚Äù) and popularity, rather than solely by genre.
  • Label clarity: The original label ‚ÄúChannels‚Äù was ambiguous. Based on user feedback, it was changed to ‚ÄúWhere to Watch‚Äù, which aligned better with user expectations.
  • Pagination issues: Participants found traditional pagination disruptive. A ‚ÄúLoad More‚Äù button was introduced to preserve continuity without overwhelming users.
  • Event structure: Users navigated events more easily when grouped by recency ‚Äî ‚ÄúUpcoming‚Äù and ‚ÄúPast‚Äù ‚Äî rather than by event type (e.g. festivals, workshops).

Recommendations

Based on IA testing and usability feedback, I developed the following actionable design improvements:

  • Replace unclear labels with user-centered terms: For example, ‚ÄúChannels‚Äù ‚Üí ‚ÄúWhere to Watch‚Äù, aligning with user expectations. All platform features were reviewed for naming clarity using card sorting insights to ensure terminology matched mental models.
  • Group events by recency, not type: Users were overwhelmed by overlapping categories like ‚ÄúWorkshops‚Äù and ‚ÄúFestivals.‚Äù Reorganizing the structure into ‚ÄúAll Events,‚Äù ‚ÄúUpcoming,‚Äù and ‚ÄúPast‚Äù improved scannability and reduced decision fatigue.
  • Adopt ‚ÄòLoad More‚Äô instead of pagination or infinite scroll: Traditional pagination disrupted browsing flow, while infinite scroll made users lose orientation. A ‚ÄúLoad More‚Äù button provided a balanced way to reveal more content without performance or cognitive strain.
  • Introduce preview-rich popups for films: Users wanted more context before clicking into a film page. I recommend adding hover-based preview cards or expandable popups to show summaries, cast, reviews, and campaign links at a glance.
  • Incorporate browsing-based recommendations: To increase engagement, future versions could surface relevant films or events based on past views, visited categories, or support history.

Outcome

This project received strong feedback for its:

  • Research depth (especially stakeholder interviews)
  • Strategic use of card sorting and tree testing to validate IA decisions
  • Clear structural diagrams and thoughtful labeling based on real user input
  • Realistic wireframes that balance discoverability with actionability

The IA and wireframe designs demonstrated how domain-specific insights can shape meaningful, user-friendly product architecture. The final report was praised for its strong process documentation and alignment with user-centered design principles.

Reflection

This project helped me develop and apply a solid information architecture methodology grounded in user research and iterative design.

What I did well:

  • Mapped a clear domain model based on real user input and competitor insights, ensuring the structure reflected actual mental models
  • Created flexible structures that allow different user types (viewers, filmmakers, backers) to achieve goals through multiple paths
  • Iterated based on feedback ‚Äî adapting terminology, grouping navigation differently, and adjusting layout styles to boost usability

What I would improve:

  • Conduct more usability testing on mid-fidelity wireframes, particularly around film filtering logic and event discovery flows
  • Prototype the donation flow to test for friction in choosing contribution amounts and establishing trust in campaign legitimacy
  • Explore accessibility affordances like keyboard navigation patterns, tab indexing, and structured alt text for information-dense pages

This project reinforced how user validation at every stage ‚Äî from naming to navigation to layout ‚Äî drives stronger digital experiences. It also reminded me that clarity and hierarchy are not just IA principles, but the foundation of inclusive, intuitive design.
